Features added in HAL driver

Added three functions to the USB20HR HAL driver (All functions are NON-Blocking)

 

1.      sls_avalon_usb20hr_config

This function configures the device structure parameters to its default state for initial usage.

 

2.      dma_usage

  Altera DMA usage can be bypassed using this function, but the transfer throughput is reduced when used without DMA

  If user writes dma_usage (0,DMA_BASE), then DMA is bypasseD and if user writes dma_usage (1, DMA_BASE), then data transfer in BULK_IN and BULK_OUT pipe done by DMA. User has to make sure that DMA should be included in SOPC builder and also connected to required memory.

 

3.      sls_avalon_usb20_connect

This function connects the USB20HR device to EHCI controller and then start enumeration. If user does not write this function then enumeration does not start.

 

Version 1.1

Known issues and Fixes

Issue#1 (Core Issue):

  Device gets time out because HOST delayed in acknowledgment on chipset ICH5 and higher chipset of Intel.  This causes instability in enumeration and transfers.

 

Fix to Software Issue#1 for USB20HR v1.0:

  To transfer the data, device must come into resume state from the suspend state. Fixed this issue.

 

Fix to Board Issue#1 for USB20HR v1.0:

  To properly detect the attach and detach conditions inside the core, added a pull down resistor of value 510 ohms to the USB vbus line on the USB B-Type device connector.

  New snap-on boards are shipped with this issue fixed.

Version 1.0

Known issues and Fixes for USB20HR v1.0

Issue#1 (Software Issue):

  Device switches from high speed mode to full speed mode when some of the windows operating system issues suspend command which causes device to suspend. Because of this, "data could not be transferred".

 

 

Issue#2 (Board Issue - applicable to USB20 Snap on Board SLS-CY-SC-USB20 only):

  Attach and Detach condition was not detected in the core because of the floating condition of the USB VBus line.
